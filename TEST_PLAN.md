# 機能テスト計画書 (Functional Test Plan)

## 1. 認証機能 (Authentication)
- [ ] **TC-001: ログインリダイレクト**
    - 手順: 未ログイン状態でトップページ (`/`) にアクセスする。
    - 期待値: `/login` にリダイレクトされること。
- [ ] **TC-002: モックログイン**
    - 手順: ログイン画面で「Mock Login」ボタンをクリックする。
    - 期待値: ダッシュボード (`/dashboard` または `/`) に遷移し、ログイン状態になること。

## 2. 試験作成機能 (Exam Management)
- [ ] **TC-003: 新規試験作成 - 正常系**
    - 手順: 
        1. 「試験作成」ページ (`/exams/new`) にアクセスする。
        2. 試験名に「テスト試験2026」を入力する。
        3. 実施日に未来の日付（例: 2026-12-31）を入力する。
        4. 「作成」ボタンをクリックする。
    - 期待値: 作成が成功し、試験詳細またはリスト画面に遷移すること。
- [ ] **TC-004: 新規試験作成 - バリデーション**
    - 手順: 空のフォームで「作成」ボタンをクリックする。
    - 期待値: エラーメッセージが表示され、作成されないこと。

## 3. AI学習ロードマップ生成 (AI Roadmap)
- [ ] **TC-005: ロードマップ自動生成**
    - 手順: 
        1. 作成した「テスト試験2026」の詳細ページを開く。
        2. 「AI学習計画を生成」ボタンをクリックする。
        3. 数秒〜数十秒待機する。
    - 期待値: 
        1. 読み込み中（Loading）の表示が出ること。
        2. 学習タスクのリスト（ロードマップ）が生成され、画面に表示されること。
        3. エラーが発生しないこと。

## 4. UI/UX・表示確認 (Display)
- [ ] **TC-006: ダッシュボード表示**
    - 手順: ダッシュボードにアクセスする。
    - 期待値: カウントダウンタイマーや次のタスクが正しく表示されていること。
- [ ] **TC-007: 試験リスト表示**
    - 手順: 試験一覧ページにアクセスする。
    - 期待値: 作成した試験が表示されていること。

## 5. 削除機能 (Deletion)
- [ ] **TC-008: 試験削除**
    - 手順: 
        1. 「テスト試験2026」の削除ボタン（ゴミ箱アイコン等）をクリックする。
        2. 確認ダイアログがあれば承認する。
    - 期待値: 
        1. 削除処理が完了すること。
        2. 一覧画面から該当の試験が消えていること。
        3. 500エラー等が発生しないこと。
